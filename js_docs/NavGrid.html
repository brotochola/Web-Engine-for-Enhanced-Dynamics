Class: NavGrid

NavGrid()

NavGrid - Navigation grid for pathfinding Provides: - Flowfield pathfinding (for masses of entities going to same target) - A* pathfinding (for individual path queries) - Grid utilities (walkability checks, cell conversions) Usage pattern: - Call requestVector() or getNextAStarPosition() every frame - If data not ready, returns fallback (0,0 or current position) - Particle worker computes in background, next frame will have data

(static) requestVector(cx, cy, tx, ty, outVec)

Get movement vector from flowfield pathfinding This is the main API for flowfield navigation. Call every frame for each entity that needs pathfinding. The system handles caching automatically.
Parameters:
Name Type Description
cx number Current world X position
cy number Current world Y position
tx number Target world X position
ty number Target world Y position
outVec Object Output vector {x, y} to fill If flowfield exists: outVec = normalized direction vector (-1, 0, or 1 per axis) If flowfield not ready: outVec = {x: 0, y: 0}, requests calculation
Source:
  • core/NavGrid.js, line 471
Example
const vec = { x: 0, y: 0 };
NavGrid.requestVector(entity.x, entity.y, target.x, target.y, vec);
entity.vx += vec.x * speed;
entity.vy += vec.y * speed;
