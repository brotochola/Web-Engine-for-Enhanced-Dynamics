Class: SpatialWorker

SpatialWorker()

SpatialWorker - Row-based spatial hashing and neighbor detection KEY INSIGHT: By partitioning grid rows across workers, we eliminate ALL race conditions without any synchronization overhead. Each worker is the sole owner of its rows - no other worker can write to them.

STEP 2: Find neighbors for all entities owned by this worker - Iterates through all owned cells - For each entity, checks if this worker owns it (based on entity's home row) - Only processes entities whose center Y falls in a row owned by this worker - Searches 3x3+ neighborhood (can read ANY cell) and writes neighbor data ENTITY OWNERSHIP: Each entity is owned by exactly ONE worker based on its "home row" (the row containing its center Y position). rebuildOwnedRows()

STEP 1: Rebuild owned rows of the spatial grid (RACE-FREE) STRATEGY: Build counts locally, then copy to grid at the end. This ensures gridCounts is never 0 during rebuild - other workers reading cells either see old data or new final data, never mid-clear. - Phase 1: Clear LOCAL counts (not grid counts!) - Phase 2: Insert entities using local counts, write entity data to grid - Phase 3: Copy local counts to gridCounts (single atomic-ish write per cell) IMPORTANT: We iterate ALL entities because an entity at any position might belong to one of our rows. But we only write to our owned cells.
